# ケトダイエット管理アプリ（PWA）

音声と写真で食事を記録し、登録した健康情報を踏まえて Claude がケトーシス（脂質燃焼）に向けたフィードバックを返すモバイル向けPWAです。ビルド不要・静的配信だけで動きます。

## 特長

- **音声で記録**: マイクボタンで「何を食べたか」を話すと文字起こし（Web Speech API）
- **写真で記録**: 食事を撮影/選択。Claude の vision で内容と栄養価を推定
- **パーソナルなAIフィードバック**: 健康プロフィール（基本身体情報・目標・血液/体組成・既往/制限）＋**今日の筋トレ・睡眠・リカバリー・健康診断値**を踏まえ、ケト視点で評価・提案・注意点を提示
- **筋トレ記録**: 実施有無＋時間＋種別をワンタップ記録。連続日数(streak)でモチベ維持
- **Oura Ring連携**: 睡眠スコア・リカバリー・HRV・安静時心拍・歩数を取り込み（手動入力／Token自動同期）
- **健康診断データ**: 血圧・HbA1c・血糖・脂質・肝機能・尿酸などを記録し、経年推移をグラフ化
- **進捗グラフ**: 体重・体脂肪の推移、筋トレ頻度、Oura回復、ネット糖質の推移をSVGチャートで可視化
- **自動栄養計算**: BMR / TDEE / 目標カロリー / ケト用PFC目標を自動算出
- **ケトーシス判定**: 血中ケトン体（β-ヒドロキシ酪酸）から状態を判定
- **AIコーチ**: 直近の食事・運動・睡眠・streakを踏まえた総合アドバイス
- **完全ローカル保存**: 健康データ・記録はこの端末（localStorage / IndexedDB）にのみ保存

## 使い方

### 1. 配信する

静的ファイルなので、HTTPSで配信すればそのまま動きます。ローカルで試すには:

```bash
cd keto-app
python3 -m http.server 8080
# ブラウザで http://localhost:8080 を開く
```

スマホで使う場合は、HTTPSの静的ホスティング（GitHub Pages / Netlify / Cloudflare Pages 等）に `keto-app/` をそのまま置いてください。マイク・カメラ・PWAインストールはHTTPS（または localhost）が必須です。

### 2. APIキーを設定する

1. [console.anthropic.com](https://console.anthropic.com/) で API キーを発行
2. アプリの「設定」タブにキーを貼り付けて保存
3. モデルは Opus 4.8（高精度）/ Sonnet 4.6（高速）/ Haiku 4.5（最安）から選択

### 3. プロフィールを入力する

「プロフィール」タブで身体情報・目標・血液データ・既往/制限を入力。入力するほどフィードバックが最適化されます。

### 4. 食事を記録する

「記録」タブで、写真を撮る／音声で説明する → 「記録・分析」。AIが栄養価を推定し、ケトスコアとフィードバックを返します。

### 5. 今日のコンディションを記録する

「ホーム」の「今日のコンディション」で、体重・体脂肪・血中ケトン・筋トレ（実施＋時間＋種別）をワンタップ記録。「進捗」タブで推移グラフと連続日数が見られます。

### 6. Oura Ring を連携する（任意）

- **手動入力**: 「ホーム」の睡眠スコア等を手で入れればすぐグラフ化されます。
- **自動同期**: [cloud.ouraring.com](https://cloud.ouraring.com/) で Personal Access Token を発行し、「設定」に貼り付け → ホームの「Ouraを同期」。
  - ⚠️ Oura API はブラウザからの直接アクセスを **CORS でブロック** します。そのため自動同期には**CORSプロキシ**が必要です。「設定」のプロキシ欄に、`Authorization` ヘッダーを通すプロキシのURL（`{url}` を含めると対象URLに置換）を指定してください。アプリ化の際はサーバー側で同期する構成に移行するのが推奨です。

### 7. 健康診断データを入力する

「設定」→「健康診断データ」で受診日と各数値を入力。「進捗」に経年の推移グラフが表示されます。

### 8. ホーム画面に追加

ブラウザの「ホーム画面に追加」でネイティブアプリのように使えます。

## 技術構成

- 依存ライブラリゼロのバニラJS（ES Modules）
- Claude API をブラウザから直接呼び出し（`anthropic-dangerous-direct-browser-access`）
- 構造化出力（`output_config.format` / JSON Schema）で栄養値を確実にパース
- Service Worker でアプリシェルをキャッシュ（オフライン起動対応）

```
keto-app/
├── index.html
├── manifest.webmanifest
├── service-worker.js
├── css/styles.css
├── icons/icon.svg
└── js/
    ├── app.js          # タブ切り替え・ルーティング
    ├── ai.js           # Claude API クライアント
    ├── oura.js         # Oura Ring v2 API クライアント
    ├── charts.js       # 依存ゼロのSVGチャート
    ├── speech.js       # 音声入力
    ├── nutrition.js    # BMR/TDEE/ケト計算・streak
    ├── store.js        # プロフィール・設定・各種定義（localStorage）
    ├── db.js           # 食事/日次/健康診断の保存（IndexedDB）
    ├── ui.js           # 共通UIヘルパー
    └── views/          # 各画面（dashboard/log/progress/history/profile/checkups/settings）
```

## セキュリティ・プライバシー上の注意

- これは**個人利用前提**のアプリです。APIキーはブラウザの localStorage に平文で保存され、リクエストヘッダーに乗ります。**共用端末では使用しないでください。**
- 食事写真・健康データは端末内（IndexedDB / localStorage）にのみ保存され、サーバーには送信されません（AI分析時に Anthropic API へ画像とテキストが送られます）。
- 本アプリは医療アドバイスではありません。持病・服薬がある場合は医師に相談してください。

## 既知の制約

- 音声認識（Web Speech API）は Chrome / Safari(iOS) で動作。非対応環境ではテキスト入力にフォールバックします。
- 栄養価はAIによる推定値です。正確な計測ではありません。
